The main refinements I’d make are: keep financial score fully deterministic, make management score partly quant and partly LLM-assisted but with hard audit trails, and make the composite weighting configurable by sector instead of fixed at 0.6/0.4 forever.

What works
Your three-layer structure is clean and matches how sector analysis should work, because general ratio analysis gives the common base while sector-specific KPIs differentiate businesses like banks, insurers, hospitals, and retailers. Keeping grades relative to sector peers is also correct, since Zerodha’s sector framework explicitly notes that no two sectors should be analyzed the same way and that peer comparison must use sector-relevant operating metrics.

Best changes
I would keep this architecture, but tighten it in five places:

Add a separate data_confidence_score, because your coverage is uneven across Screener, IndianAPI, and management inputs, and users should see whether a rank is based on full or partial evidence.

Split management into management_quant_score and management_llm_score, so the LLM is a contributor rather than the whole layer.

Make composite weights sector-configurable, because sectors with strong hard KPIs like banks may deserve more quant weight than sectors where execution commentary matters more.

Add minimum peer-count rules, because percentile ranking becomes noisy in tiny sub-sectors.

Add “coverage-aware renormalization,” so missing sector KPIs do not automatically punish a stock when the data source simply lacks them.

Management layer
This is the part most likely to drift, so I’d make it very explicit that the LLM does not “judge management” freely; it scores against a fixed rubric. For each stock, the LLM should receive only structured evidence, return sub-scores plus a reason, and every reason should map to a cited input item such as guidance delivered, capex completed, pledge unchanged, or cash conversion deterioration.

A clean rubric is:

Execution delivery.

Capital allocation.

Governance/alignment.

Accounting conservatism.

Disclosure quality.

Then define a hard cap rule: if key quant red flags exist, the LLM cannot assign a high overall management score even if commentary sounds strong.

Ranking logic

I’d add one more output field—sector_gap_to_leader—because it tells you whether rank 2 is almost tied with rank 1 or far behind, which is often more informative than the letter grade alone.

Coverage rules
Given your coverage numbers, you should define hard operating rules before moving to sector KPIs.

Recommended rules:

If Screener and IndianAPI both exist, use blended metric source with source priority by metric family.

If only Screener exists, compute everything possible from raw statements and mark derived coverage.

If only IndianAPI exists, use keyMetrics but reduce confidence if raw historical statements are missing.

If sector classification is missing, assign sector = UNMAPPED and exclude from sector ranking until mapped.

If peer count in a sector is below 5, rank but mark low_peer_count = true.

If metric coverage for a company is below a threshold, still score using renormalized available weights, but show low confidence.

Those rules make the score usable instead of silently biased.
