## Recommendation

The feature library needs rethinking at the **data layer** and
**hurdle layer** before running another pilot:

- **Data layer:** the 252-bar features need a panel that pre-dates
  TRAIN_VAL_START by at least 252 trading days on every ticker —
  including pseudo-tickers if any feature derives from them.
  Otherwise fold 0 is always empty on those features and the pilot
  wastes budget re-proposing them.
- **Hurdle layer:** scarcity fallback to NEUTRAL NIFTY B&H is the
  wrong benchmark for a cross-sectional long-short basket. A
  regime-conditional basket B&H (equal-weight long top-k minus
  bottom-k on a neutral feature) would be the honest hurdle. The
  current hurdle rewards bearish tilts mechanically.
- **Feature library:** we have not tested a single positive-Sharpe
  rule. The library's 20 features + 4 constructions × 8 thresholds
  grid has been sampled sparsely by Haiku (22 rows); of those, only
  the survival of 2 degenerate cases suggests we need to widen the
  search (autonomous mode) or re-ground the features themselves.

A re-proposal round with these fixes, before promoting anything to
holdout, is the minimum next step.
