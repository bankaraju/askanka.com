Unified Analysis Panel — what shape are we building?

Terminal currently: Spreads renders a 5-layer drawer, FCS adds a mini-panel underneath, Correlation Breaks lives on Scanner/Regime pages, TA is a design on paper. They do not share a visual template — so the terminal "feels broken." Two possible unifications:

A

Shared anatomy, same surfaces

Keep current locations. Spreads stay in candidate drawer. FCS stays in drawer. TA gets the same drawer treatment. Correlation Breaks keep their Scanner/Regime pages. But every analysis — wherever it lives — renders as the same component template: header band → verdict tile → contribution bars → health strip. One CSS file, one JS shape, four consumers.

Smaller lift. Feels coherent because the micro-layout repeats. No routing or tab changes.

B

Convergence point — "Ticker Brief"

New location. One dedicated panel per ticker that stacks ALL four analyses (Spread presence, TA verdict, FCS score, Correlation Break status) in a single card. Opened from any row click, any page. The spreads tab, scanner, trading tab all point to this one brief.

Bigger lift. Feels coherent because there is literally one place. But it collapses trade-centric (spreads) and ticker-centric (TA/FCS) into one — needs a schema reconciliation first.

C

Both — shared anatomy AND a ticker brief

Build A first (cheap, immediate cohesion). Then in v2 introduce the Ticker Brief as a roll-up view that reuses the same anatomy. The brief becomes an additional surface, not a replacement.

Two phases, but v1 ships value in days, not weeks.
