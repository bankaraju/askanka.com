Scanner Layout: How should the filter + results look?

The Scanner is a new sub-tab in Trading. Filter → list → click → detail. Three layout approaches:

A

Filter Bar + Table

Spreadsheet-style — dense, scannable, sortable

Click row → loads stock in Charts + TA

Pros

  • Most information density
  • Sortable by any column
  • Familiar spreadsheet feel

Cons

  • Dry — no visual weight to strong signals
  • Every row looks the same importance
B

Filter Bar + Card Grid

Visual cards — grouped by stock, patterns as badges inside

Pros

  • Grouped by stock — see all patterns at once
  • Visual weight — stocks with more patterns stand out
  • Compact stat lines per pattern

Cons

  • Harder to sort across stocks
  • Cards take more space than table rows

MY RECOMMENDATION: B (Card Grid)

The table (A) is familiar but treats every pattern-stock combination as equal — 47 flat rows. The card grid (B) groups by stock, which is how you'll actually use it: "I want to research RELIANCE" not "I want to research ATR_COMPRESSION." The compact stat line you approved (A from the previous question) fits perfectly inside each card.

Also: cards match the terminal's existing design language (spread cards, trust score cards, correlation break cards).
