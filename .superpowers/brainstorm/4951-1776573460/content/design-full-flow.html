4. Backend: New API Endpoint

GET /api/scanner?min_win=70&direction=ALL&min_occ=10&sort=win_rate
Reads all 213 fingerprint files from /data/ta_fingerprints/
Filters patterns by win_rate, direction, significance, occurrences
Groups by stock, sorts by best pattern's metric
Returns: { stocks: [{symbol, patterns: [...], best_win, pattern_count}], total_stocks, total_patterns }
Single endpoint. No new data files needed — reads existing TA fingerprints. Filtering happens server-side to keep the response lean.

SCOPE SUMMARY

Build:
· 1 new API endpoint: /api/scanner
· 1 new sub-tab: Scanner (first position in Trading)
· Shared ticker state across Charts/TA/Scanner sub-tabs
· Compact stat narration per pattern (template, not LLM-generated)

Don't build:
· No spread filtering (stays in Spreads tab as-is)
· No LLM-generated narration (just formatted stats)
· No new data pipeline (reads existing fingerprints)
· No changes to Signals or Spreads sub-tabs
