A: Interactive Explorer

Kill the grid. Replace with clickable event pills at the top. When you click an event (e.g., "HORMUZ"), it reveals ranked spread cards showing which trades historically worked best. The heatmap becomes a story you explore, not a spreadsheet you decode.

Pros

  • Instantly understandable
  • Mobile-friendly
  • Tells a narrative
  • Uses all the rich data (spread_1d, spread_5d, n)

Cons

  • Loses the "all data at once" overview
  • More JS complexity

B: Focused Grid + Spotlight

Keep the grid but show only top 10 spreads by default. Below the compact grid, add a "Spotlight" callout highlighting the strongest active setup. Full 25-row grid available via "Show All" toggle. Less noise, same data.

Pros

  • Familiar grid format
  • Much less overwhelming
  • Spotlight adds narrative
  • Easy to implement

Cons

  • Still a grid at heart
  • Column headers still rotated

C: Hybrid — Explorer + Grid Toggle

Best of both worlds. Default view is the Explorer (event list on left, ranked results on right). Tab to "Full Grid" for the complete heatmap when you want the raw data. The explorer becomes the hero; the grid becomes the reference appendix.

Pros

  • Explorer is the wow moment
  • Full grid still accessible
  • Shows spread_5d, n — richer than just %
  • Progress bars add visual punch

Cons

  • Most complex to build
  • Two views to maintain

My recommendation: C (Hybrid) — the Explorer gives you the wow factor, while the Grid tab keeps the data purists happy. The Gamma-style clean fonts and flow you liked can be applied to the Explorer view perfectly.
