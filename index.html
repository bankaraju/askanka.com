How We Know What We Know

Methodology

Anything can look like signal in-sample. We constrain ourselves to the few practices that actually distinguish edge from luck.

Pre-registration

Every hypothesis is written and committed to a public registry before data is touched. Spec specifies the rule, the universe, the holdout window, and the pass/fail thresholds. No fishing, no post-hoc story-telling.

Single-touch holdout

Each hypothesis gets one out-of-sample window. Results are judged once. No re-running with adjusted parameters after a fail — that's the difference between research and rationalisation.

Label-permutation null

Returns are permuted 10,000× to build the no-skill distribution. We require α = 0.05 against this null on the pre-specified statistic, not against a normal-theory approximation that almost never holds.

Comparator baselines

A signal must beat random direction, always-prior, long-only, and a regime-conditional baseline — not just a coin flip. Most "edges" disappear once the right baseline is built.

Data validation gate

No backtest may run on data that hasn't been registered, schema-checked, point-in-time-corrected, and contamination-mapped. The data policy is a hard prerequisite for any model decision.

Strategy registration kill-switch

Any new trading-rule file in the codebase requires a matching registry entry in the same commit. Enforced by a pre-commit hook and a CI gate. No untraceable strategies, ever.
